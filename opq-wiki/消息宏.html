# 消息宏介绍

所谓的消息宏类似其他框架的xx码个人觉得定义为宏更确切一些,肯定是在运行时解析替换的东西,也可叫做文本宏,通过发送文本达到内部替换进而实现具体的功能

URL: http://127.0.0.1:8888/v1/LuaApiCaller

method :POST

# 框架支持的宏:
  • [表情182] 表情码通过拦截消息所得 支持多表情码嵌套 暂时没整理所有的表情代码
  • [PICFLAG] 控制图文的顺序默认先图后文字在文字里插入此宏则改变为先文字后图现实
  • [秀图40001] 范围 40000-40006 实现秀图发送 群有有效好友无效
  • [GETUSERNICK(QQ)] 取某个QQ昵称 通过网络获取 不支持多个QQ 如获取QQ号为123456789的昵称 则插入发送文本消息[GETUSERNICK(123456789)]
  • [ATALL()] @所有人
  • [ATUSER(QQ1,QQ2,QQ3)]@某人 支持多个QQ以此类推如ATQQ为123和456则插入发送文本[ATUSER(123,456)] 如ATQQ为112233445566则插入发送文本[ATUSER(112233445566)]
  • [吃货],[哇啊~],[喵喵拳]等部分动态表情 可以通过消息拦截到
# WebAPi请求参数:

params:

{
  'qq':123456, #机器人QQ号
  'funcname':SendMsgV2, #调用方法类型
}

以发送群消息为例 body(json):

# 发送小表情
{"ToUserUid":123456789,"SendToType":2,"SendMsgType":"TextMsg","Content":"[表情182]HelloWorld[表情182]"}

# 发送先文字后图片 在文本任意位置插入宏 [PICFLAG] 即可
{"ToUserUid":757360354,"SendToType":2,"SendMsgType":"PicMsg","PicPath":"./demo.jpg","Content":"HelloWorld[PICFLAG]"}

# 发送秀图 图片demo将以秀图方式展示
{"ToUserUid":757360354,"SendToType":2,"SendMsgType":"PicMsg","PicPath":"./demo.jpg","Content":"HelloWorld[秀图40006]"}

# 获取123456789的昵称
{"ToUserUid":123456789,"SendToType":2,"SendMsgType":"TextMsg","Content":"[GETUSERNICK(123456789)]HelloWorld[表情182]"}

# AT 12345678
{"ToUserUid":123456789,"SendToType":2,"SendMsgType":"TextMsg","Content":"[ATUSER(123456789)]HelloWorld[表情182]"}

# AT全体成员
{"ToUserUid":123456789,"SendToType":2,"SendMsgType":"TextMsg","Content":"[ATALL()]"}

# LuaApi发送XML/JSON

⚠️我就不一一复制发送结构了 WebApi的Json 对应Lua里的table 映射规律是这样的 把Json的字段的引号都去掉 字段值的冒号换成=号 数组[]换成{}⚠️

if data.Content == "SendMsgV2Test" then
        Api.Api_SendMsgV2( --私聊
            CurrentQQ,
            {
                ToUserUid = 987654321,
                SendToType = 2,
                SendMsgType = "TextMsg",
                Content = "[表情182]HelloWorld[表情182]"
            }
        )
    end
